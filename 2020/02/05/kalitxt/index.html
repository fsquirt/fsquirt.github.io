解决新版kali中文乱码和不能使用dpkg装软件

发表于 2020-02-05 更新于 2020-03-13

本人是个linux刚入门的新手，文章可能有误或存在争议，反馈邮箱fsquirt@qq.com，非常感谢支持！

中文乱码

这两天想玩黑客，就从kali的官网上把镜像下了过来，安装界面改版了，UI也改版了
但我进桌面就一脸懵逼，怎么，中文全是乱码？？

解决方法

Step1.设置root的密码
你没听错，新版的kali不是默认以root进行登录的，所以打开终端输入：
1.sudo passwd root
2.输入你新设置的账户的密码
3.把你想设置的root的密码输进去
4.再输一遍来确定

Step2.加源
注销，使用root登录，打开**/etc/apt/sources.list** ，在文末加上:

#阿里云
deb http://mirrors.aliyun.com/kali kali-rolling main non-free contrib
deb-src http://mirrors.aliyun.com/kali kali-rolling main non-free contrib

保存并退出，不用退出root

Step3.安装中文字体
打开终端，输入

1.apt-get install xfonts-intl-chinese
2.apt-get install ttf-wqy-microhei

安装中文字体

Step4.能干嘛？重启呀！

进桌面后，中文成功显示

不能dpkg装软件

听说QQ有linux版了？把deb包下载来，丢进kali里，sudo dpkg -i QQ.deb,然后报错了：

dpkg: 警告: 在 PATH 环境变量中找不到 ldconfig 或没有可执行权限
dpkg: 警告: 在 PATH 环境变量中找不到 start-stop-daemon 或没有可执行权限
dpkg: 错误: 2 在环境变量 PATH 中找不到该程序，或不可执行
提示：root 的 PATH 环境变量通常应当包含 /usr/local/sbin、/usr/sbin 和 /sbin

解决方法

输入 export PATH=/usr/loca/sbin:/usr/sbin:/sbin:$PATH
然后再dpkg -i QQ.deb，linux版QQ就装好了

顺带说句，使用linux版QQ仿佛使我回到了十年前
